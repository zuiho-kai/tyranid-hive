### 记录规则

- **条目结构**：❌ 错误做法 + ✅ 正确做法，整条控制在 **5 行以内**
- **❌/✅ 写法**：只写核心规则，禁止多层加粗嵌套（加粗最多一层）
- **`>` 注释**：只放归因和 postmortem 链接，禁止复述场景
- **复犯**：只追加频率标记（🟡×N / 🔴×N）
- **归属**：条目只写本范围内容，跨范围写到对应文件

### DEV-90 语法检查/读代码冒充真实验证 `🟡×2`

❌ 用 `ast.parse` / `node -e "new Function(...)"` 等纯语法检查代替"实际运行验证"，发现环境不符就降级验证标准
❌ 读代码确认"逻辑链路正确"就宣布验证通过——人肉 trace 不是验证，是自说自话
✅ 必须用项目要求的 Python 版本在真实环境中导入模块、验证端点注册。本地版本不符时用 `py -3.13` 或创建临时 venv，不能降级
✅ 结单自验证 = 写测试脚本实际运行，mock 最小依赖，断言预期行为。"我看了代码没问题"不算验证
