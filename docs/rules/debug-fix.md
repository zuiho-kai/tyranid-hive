# 硬规则 — 调试与修复

- **两次失败必须搜索**（COMMON-9）：连续猜方案失败两次后停下搜索根因
- **两次补丁失败必须回退换路**（DEV-81）：连续 2 次补丁未解决 → 停下，列出链路副作用，回退到最后正常状态，换完全不同的链路
- **平台 API 先最小复现**（DEV-82）：涉及平台 API 异常行为时，第一步建最小可独立复现的示例
- **3 次 fix commit 强制停下**（DEV-83）：同一功能累计 3 次 fix commit 未解决 = 强制停下 → 回退到第一次 fix 前 → 换技术路径
- **平台能力边界先确认**（DEV-84）：使用平台 API 前：①查官方文档 ②搜 GitHub Issues ③最小复现验证
