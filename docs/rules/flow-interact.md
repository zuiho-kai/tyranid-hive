# 硬规则 — 流程与交互

- **提问即交权**（DEV-53）：同一轮消息禁止"提问 + 执行"并存。提问后唯一允许的动作是等用户回答
- **防丢失**：对话是临时的，文件是持久的（COMMON-1）
- **不做口头承诺**（DEV-76）：教训/流程改进发现当下直接写入对应错题本文件 + 更新 `_index.md`。口头说了但没落盘 = 没做（COMMON-12）
- **设计即落盘**（DEV-87）：Module 开发流程每一步（Step 1-4）的输出必须在完成当步时立即落盘到 `docs/design/<module>/` 对应文件，不允许只在对话中输出而不写文件
- **禁止脑补**：所有操作必须基于已确认的信息。抓取/搜索失败时不要用猜测的内容代替，必须先确认再写
- **禁止无证据的"根治"**（DEV-85）：禁止在 commit message 中使用"根治/彻底修复/完美解决"等词，除非附带验证证据
